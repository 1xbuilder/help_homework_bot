# Homework Bot — городская версия

Телеграм-бот для домашних заданий. Один бот обслуживает много институтов и
групп: каждый видит и редактирует ДЗ только своей группы. Присоединение — по
пригласительным ссылкам, права — по иерархии ролей.

## Иерархия ролей
- **admin** (ты) — всё, везде. Задаётся переменной `ADMIN_IDS`.
- **moderator** — глобальный помощник админа.
- **owner** (староста) — админ своей группы.
- **helper** (помощник старосты) — добавляет/удаляет ДЗ.
- **member** — обычный пользователь, только смотрит.

Глобальные роли (admin/moderator) хранятся в `users.global_role`.
Роли внутри группы (owner/helper/member) — в `group_members`, поэтому один
человек может быть старостой одной группы и обычным членом другой.

## Два типа пригласительных ссылок
1. **create_group** — одноразовая. Ты выдаёшь старосте (`/admin` в боте).
   По ней он создаёт группу и становится её владельцем.
2. **join_group** — многоразовая. Староста создаёт в своей админке (`/group`)
   и кидает в чат группы — одногруппники входят как обычные участники.

## Запуск с нуля

### 1. База данных (Supabase)
1. Создай проект на https://supabase.com (или используй существующий).
2. Открой **SQL Editor**.
3. Выполни `migrations/001_init_schema.sql` (создаёт все таблицы;
   **внимание**: дропает старые `users`/`homeworks`).
4. Выполни `migrations/002_seed.sql` (заводит первое заведение — ОмГТУ).
5. В **Project Settings → API** скопируй `Project URL` и `service_role` ключ.

### 2. Бот в Telegram
1. Напиши @BotFather → `/newbot`, получи токен вида `123456:ABC...`.
2. Узнай свой Telegram ID у @userinfobot (число вроде `123456789`).

### 3. Переменные окружения
Создай файл `.env` в корне проекта (или задай переменные на хостинге):
```
BOT_TOKEN=123456:ABC...
SUPABASE_URL=https://xxxx.supabase.co
SUPABASE_KEY=<service_role ключ>
ADMIN_IDS=123456789
```
Несколько админов — через запятую: `ADMIN_IDS=111,222`.

### 4. Установка и запуск
```
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
python main.py
```
Должно появиться: `✅ Подключено к Supabase через REST API` и `Бот запущен...`.

## Первый прогон (проверь, что всё работает)
1. Напиши боту `/start` → зарегистрируйся (имя).
2. Напиши `/admin` → откроется твоя панель (роль admin придёт из `ADMIN_IDS`).
3. «Выдать ссылку на создание группы» → выбери ОмГТУ → бот пришлёт ссылку.
4. Перейди по этой ссылке сам (или дай старосте) → введи название группы
   (напр. ИС-301) → ты станешь старостой, выберешь подгруппу.
5. Напиши `/group` → «Ссылка-приглашение» → пришлёт ссылку для одногруппников.
6. «Участники» → можно назначать помощников.
7. В главном меню: «➕ Добавить ДЗ» (как староста/помощник), «📚 Посмотреть ДЗ».
   Обычный участник при попытке добавить/удалить получит отказ.

## Структура проекта
```
main.py                  — точка входа, регистрация хендлеров
config.py                — переменные окружения (TOKEN, SUPABASE, ADMIN_IDS)
migrations/              — SQL для Supabase (схема + сид + README)
database/
  models.py              — ORM-модели (описание схемы)
  db_session.py          — подключение к Supabase
  db_operations.py       — весь слой работы с БД (CRUD, роли, инвайты)
  codes.py               — генерация пригласительных кодов
handlers/
  start.py               — /start, онбординг по ссылкам, регистрация
  homework.py            — просмотр ДЗ (по активной группе)
  add_homework.py        — добавление ДЗ (+ расписание по группе, права)
  delete_homework.py     — удаление ДЗ (+ права)
  calendar_handlers.py   — выбор даты в календаре
  admin_panel.py         — админки старосты (/group) и глобальная (/admin)
  admin_handlers.py      — тексты раздела «Помощь»
states/                  — состояния FSM
keyboards/, utils/       — клавиатуры и вспомогательное
MIGRATION_PLAN.md        — план переделки и что сделано по этапам
```
