# Веб-админка StudyFlow — установка

Полноценная панель управления в браузере: пользователи (ники, роли), группы,
заведения, задания, ссылки-приглашения. Логин по паролю. Service-ключ Supabase
остаётся на сервере — в браузер не попадает.

## Из чего состоит

- **`admin_auth.py`** — логин/пароль из env + подписанные токены сессии (без сторонних библиотек).
- **`admin_api.py`** — REST API под `/api/admin/*`, переиспользует `database/db_operations.py`.
- **`photo_proxy.py`** — уже existing HTTP-сервер бота; в него добавлена строка, регистрирующая роуты админки.
- **`admin.html`** — сам сайт (одним файлом). Хостится на GitHub Pages.
- **`make_password.py`** — утилита для генерации пароля.

Бэкенд крутится внутри того же процесса бота на bothost — отдельный сервер не нужен.

## Шаг 1. Сгенерируй пароль (локально)

```bash
python make_password.py
```

Введёшь логин и пароль — получишь три строки. Скопируй их.

## Шаг 2. Пропиши переменные окружения на bothost

К уже существующим (`BOT_TOKEN`, `SUPABASE_URL`, `SUPABASE_KEY`, `ADMIN_IDS`) добавь:

```
ADMIN_PANEL_LOGIN=admin
ADMIN_PANEL_PASSWORD_HASH=pbkdf2_sha256$200000$....$....
ADMIN_PANEL_SECRET=<длинная случайная строка из утилиты>
```

Перезапусти бота. В логах появится `🛠  Admin API routes registered`.

## Шаг 3. Узнай публичный URL бота

Это тот же домен, что ты вставлял (или собирался вставить) в `index.html` как
`SCHEDULE_API`. Проверить, что сервер жив:

```
https://<твой-бот>.bothost.ru/health   →  должно вернуть «ok»
```

## Шаг 4. Настрой и выложи admin.html

1. Открой `admin.html`, найди строку:
   ```js
   const API_BASE = localStorage.getItem('sf_api_base') || 'ВСТАВЬ_URL_БОТА/api/admin';
   ```
   Замени `ВСТАВЬ_URL_БОТА` на свой домен, например:
   ```js
   const API_BASE = localStorage.getItem('sf_api_base') || 'https://мойбот.bothost.ru/api/admin';
   ```
2. Залей `admin.html` в git-репозиторий (можно в тот же, где `index.html`).
3. Включи **GitHub Pages** (Settings → Pages → ветка `main`, папка `/root` или `/docs`).
4. Открой `https://<твой-логин>.github.io/<репо>/admin.html` — увидишь экран входа.

> CORS на сервере уже разрешает любой домен `*.github.io` и `localhost`.
> Если выложишь на свой домен — добавь его в `ALLOWED_ORIGIN_SUFFIXES` в `admin_api.py`.

## Шаг 5. Войди

Логин и пароль из шага 1. Токен живёт 12 часов, потом попросит войти заново.

## Безопасность — коротко

- Пароль хранится только как pbkdf2-хэш в env, в коде и репозитории его нет.
- В браузер уходит лишь подписанный токен; подделать без `ADMIN_PANEL_SECRET` нельзя.
- Все операции записи в базу идут через сервер с проверкой токена. Anon/ service
  ключи Supabase в `admin.html` отсутствуют.
- Если `ADMIN_PANEL_*` не заданы — вход заблокирован (а не открыт всем).

## Что осталось от старой админки в боте

Команды `/admin` и `/group` внутри бота продолжают работать как раньше. Когда
привыкнешь к веб-панели — их можно будет убрать, но это не обязательно.
