## ✨ Возможности

- 🏫 **Много институтов и групп** — каждый пользователь привязан к своей группе и видит только её ДЗ.
- 🔗 **Приглашения по ссылкам** — староста создаёт группу по ссылке от админа и зовёт одногруппников своей ссылкой.
- 👥 **Иерархия ролей** — админ, модераторы, старосты, помощники, обычные участники.
- 📅 **Расписание** — подтягивается из API института (где есть), иначе предмет вводится вручную.
- 📎 **Вложения** — фото, документы, видео, аудио к заданиям.

## 🧩 Роли

| Роль | Где задаётся | Что может |
|------|--------------|-----------|
| `admin` | переменная `ADMIN_IDS` | всё; выдаёт старостам ссылки на создание групп |
| `moderator` | глобально | помогает админу |
| `owner` (староста) | при создании группы | управляет своей группой, зовёт людей, назначает помощников |
| `helper` (помощник) | назначает староста | добавляет и удаляет ДЗ |
| `member` | по умолчанию | смотрит ДЗ своей группы |

## 🔗 Два типа ссылок

1. **create_group** — одноразовая. Админ выдаёт старосте (`/admin` в боте). По ней создаётся группа.
2. **join_group** — многоразовая. Староста создаёт в своей админке (`/group`) и кидает в чат группы.

## 🚀 Установка

### 1. База данных (Supabase)
1. Создай проект на [supabase.com](https://supabase.com).
2. В **SQL Editor** выполни по очереди:
   - `migrations/001_init_schema.sql` — создаёт таблицы;
   - `migrations/002_seed.sql` — добавляет первое заведение.
3. В **Project Settings → API** скопируй `Project URL` и `service_role` ключ.

### 2. Бот в Telegram
- Создай бота у [@BotFather](https://t.me/BotFather), получи токен.
- Узнай свой Telegram ID у [@userinfobot](https://t.me/userinfobot).

### 3. Переменные окружения

| Переменная | Пример | Описание |
|------------|--------|----------|
| `BOT_TOKEN` | `123456:ABC...` | токен от BotFather |
| `SUPABASE_URL` | `https://<id>.supabase.co` | URL проекта (**Project ID**, не имя!) |
| `SUPABASE_KEY` | `eyJ...` | `service_role` ключ |
| `ADMIN_IDS` | `123456789` | твой Telegram ID (несколько — через запятую) |

> На хостинге переменные задаются в панели. Локально — в файле `.env` (см. `.env.example`).

### 4. Запуск
```bash
pip install -r requirements.txt
python main.py
```
В логах появится `Подключено к Supabase` и `Бот запущен`.

## 📂 Структура

```
main.py              — точка входа, регистрация хендлеров
config.py            — чтение переменных окружения
migrations/          — SQL для Supabase
database/            — модели, подключение, операции с БД, генерация кодов
handlers/            — логика: старт, ДЗ, добавление, удаление, календарь, админки
states/              — состояния FSM
keyboards/, utils/   — клавиатуры и вспомогательное
```

## 🗺 Первый запуск по шагам

1. Напиши боту `/start`, зарегистрируйся.
2. `/admin` → выдай себе ссылку на создание группы.
3. Перейди по ней → введи название группы → ты староста.
4. `/group` → создай ссылку-приглашение для одногруппников.
5. Добавляй и смотри ДЗ.
