# Миграции БД (Supabase)

Применять по порядку в Supabase → SQL Editor.

## 001_init_schema.sql
Создаёт всю схему с нуля. **Дропает старые таблицы** `users` и `homeworks`
(тестовые данные не сохраняются — по согласованию). Просто вставь содержимое
в SQL Editor и нажми Run.

Таблицы: institutions, groups, users, group_members, invites, homeworks.

## 002_seed.sql
Начальное наполнение. Заводит первое заведение (ОмГТУ). Ничего подставлять
не нужно. Можно запускать как есть.

### Про админов
Админы задаются НЕ в SQL, а переменной окружения `ADMIN_IDS` (числовые
Telegram ID через запятую, напр. `ADMIN_IDS=123456789,987654321`). При первом
`/start` бот сам выдаст роль `admin` любому, чей ID там есть. Сменить/добавить
админа = поменять переменную и перезапустить бота. Свой ID узнай у @userinfobot.

## Порядок
1. Run 001_init_schema.sql
2. Run 002_seed.sql (без правок).
3. Прописать переменные окружения бота: BOT_TOKEN, SUPABASE_URL, SUPABASE_KEY,
   ADMIN_IDS (свой Telegram ID).
4. Поднять бота, написать ему /start, затем /admin — выдать ссылку старосте.
